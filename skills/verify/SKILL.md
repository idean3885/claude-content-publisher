# 포스팅 검증 스킬

> 포스팅 전에 **게시 가능 레벨**을 확보하기 위한 전용 검증.

---

## 호출 방식

- `/blog:verify` 직접 호출
- 트리거: "포스팅 검증", "글 검증", "verify post"

## 의존성

없음. 독립 실행 가능.

---

## 동작 흐름

### Phase 1: 대상 파악

검증 대상 포스팅 내용을 수집한다.

**입력 유형:**
- 대화 중 작성된 포스팅 본문 (가장 일반적)
- 마크다운 파일 경로
- 페이지 ID (외부 소스에서 조회 시)

대상이 불명확하면 질문한다:
```
검증할 포스팅을 알려주세요:
- 방금 작성한 내용을 검증할까요?
- 파일 경로를 알려주세요
```

### Phase 2: 3축 검증 실행

---

#### 축 1: 사실 정확성

> "기술적 주장이 정확한가?"

**검증 포인트:**
- 기술 용어가 정확하게 사용되는가?
- 코드 예시가 실제 동작과 일치하는가?
- 수치, 버전, 설정값이 정확한가?
- 인과관계 설명이 논리적으로 타당한가?
- 출처가 필요한 주장에 근거가 있는가?

**관찰 방법:**
- 기술적 주장을 하나씩 검토
- 코드 블록이 있으면 문법/로직 확인
- "~때문에", "~이므로" 등 인과 연결이 정확한지 확인

**산출물:** 사실 정확성 평가 — 확인 필요한 주장 목록

---

#### 축 2: 구조와 완성도

> "읽는 사람이 오해 없이 끝까지 읽을 수 있는가?"

**검증 포인트:**
- TL;DR이 본문 핵심을 정확히 요약하는가?
- 도입부가 독립적으로 읽히는가? (시리즈 맥락 없이 이해 가능)
- 논리 흐름이 자연스러운가? (문제->원인->해결->결과)
- 결론이 도입부의 약속을 이행하는가?
- 실무 사례가 포함되어 있는가?
- 용어가 일관되게 사용되는가?

**관찰 방법:**
- TL;DR <-> 결론 대조
- 도입부만 읽고 "이 글이 무엇을 다루는지" 알 수 있는지 판단
- 섹션 간 전환이 자연스러운지 확인
- 대상 독자 관점에서 전제 지식 요구 수준 평가

**산출물:** 구조 완성도 평가 — 가독성, 완전성, 독립성

---

#### 축 3: 독립 가치

> "이 글 하나만 읽어도 가치가 있는가?"

**검증 포인트:**
- 시리즈의 다른 편을 읽지 않아도 이 글이 성립하는가?
- 하나의 문제->해결 스토리가 완결되는가?
- 독자가 이 글에서 하나 이상의 실무 인사이트를 얻는가?
- 제목이 글의 핵심 가치를 드러내는가?

**관찰 방법:**
- "시리즈 첫 글로 이 편을 읽는 독자" 관점에서 검토
- 다른 편 참조 없이 이해되지 않는 문장 식별
- 글의 핵심 takeaway를 한 문장으로 요약 시도

**산출물:** 독립 가치 판정 — 단독 게시 적합 여부

---

### Phase 3: 결과 제시

```markdown
## 포스팅 검증 결과

**대상**: {포스팅 제목}
**시리즈**: {시리즈명} (해당 시)

---

### 1. 사실 정확성

**상태**: {확인 필요 / 양호}

{관찰 사항}

---

### 2. 구조와 완성도

**상태**: {확인 필요 / 양호}

{관찰 사항}

---

### 3. 독립 가치

**상태**: {확인 필요 / 양호}

{관찰 사항}

---

### 종합 판정

**게시 가능 여부**: {가능 / 수정 후 가능 / 보류}

{종합 소견 및 수정 권고}
```

### Phase 4: 수정 협의

검증 미통과 항목이 있으면:
1. 수정 방안을 제안한다
2. 사용자 확인 후 본문을 수정한다
3. 수정된 부분만 재검증한다

---

## 핵심 원칙

### 하지 않는 것
- 대외비 제거 (오케스트레이션 스킬의 역할)
- 코드/설계 검증 (개발 검증 도구의 역할)
- 맞춤법/문법 교정 (별도 도구 영역)
- 블로그 메타(카테고리/태그) 검증

### 하는 것
- 기술적 사실의 정확성 확인
- 글 구조의 논리적 완성도 평가
- 시리즈 독립성 판정
- 게시 가능 레벨 판정
