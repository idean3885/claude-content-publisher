# 블로그 퍼블리시

클린 콘텐츠를 Jekyll(Chirpy 테마) 블로그 포스팅으로 변환합니다.

> **역할 한정**: 이 스킬은 **형식 변환 + 파일 생성**만 수행합니다.
> 콘텐츠 검증, 대외비 제거 등은 콘텐츠 소스 측에서 완료된 상태여야 합니다.

## 트리거

- "블로그 변환", "블로그에 올려", "퍼블리시"
- "blog-publish", "publish"

## 설정

이 스킬은 **어떤 디렉토리에서든** 실행 가능합니다.
블로그 레포 경로는 글로벌 설정에서 로드합니다.

```
설정 파일: ~/.claude/blog-publisher.json
```

```json
{
  "blogRepo": "/path/to/blog-repo"
}
```

설정 파일이 없으면 사용자에게 경로를 질문하고 자동 생성합니다.

## 입력

| 항목 | 필수 | 설명 |
|------|------|------|
| 본문 | O | 정제된 마크다운 (대외비 제거 완료 상태) |
| 제목 | O | 포스팅 제목 |
| 카테고리 | O | `[메인, 서브]` |
| 태그 | O | `[태그1, 태그2, ...]` |
| 발행 날짜 | - | 미지정 시 오늘 날짜 |

입력은 다음 방식으로 제공받습니다:
- 다른 스킬에서 넘겨받은 콘텐츠 (오케스트레이션 패턴)
- 사용자가 직접 본문과 메타를 전달
- 파일 경로로 전달

## 시리즈 인덱스 포스팅

시리즈 상위 페이지는 블로그에서 **인덱스 포스팅(0번째 글)** 역할을 합니다.
이력서/포트폴리오에서 이 인덱스 포스팅 1개를 링크합니다.

| 항목 | 인덱스 포스팅 | 개별 포스팅 |
|------|-------------|-----------|
| 목적 | 프로젝트 전체 파악 (30초) | 특정 문제→해결 깊이 |
| 독자 | 면접관 (랜딩 페이지) | 관심 있는 기술 영역 독자 |
| 내용 | 프로젝트 요약 + 각 편 핵심 1줄 + 링크 | 완결된 문제→해결 스토리 |
| 갱신 | 시리즈 현행화 시 합본 역할 | 개별 수정 |

## 절차

### 1. 설정 로드

`~/.claude/blog-publisher.json`에서 블로그 레포 경로를 로드합니다.
설정이 없으면 사용자에게 질문 후 자동 생성합니다.

### 2. 블로그 규칙 참조

블로그 레포의 `CLAUDE.md`에서 포스팅 규칙(front matter, 스타일링 등)을 읽습니다.

### 3. 본문 변환

| 항목 | 입력 | 블로그 |
|------|------|--------|
| 제목 | 본문 내 `# 제목` 또는 메타 | front matter `title`로 이동, 본문에서 제거 |
| TL;DR | `## TL;DR` 섹션 | `> **TL;DR** 내용` + `{: .prompt-tip }` 로 변환 |
| 메타 섹션 | 하단 메타 블록 (있으면) | 제거 (front matter로 이동) |

#### front matter

```yaml
---
title: "{제목}"
date: {날짜} {시간} +0900
categories: [{메인 카테고리}, {서브 카테고리}]
tags: [{태그}]
description: "{TL;DR 첫 문장}"
---
```

### 4. 미리보기 및 확인

변환된 전체 포스팅을 사용자에게 보여주고 확인을 받습니다.

### 5. 파일 생성

블로그 레포의 `_posts/` 에 파일을 생성합니다.

- 파일명: `{YYYY-MM-DD}-{slug}.md`
- slug: 제목에서 키워드 추출, kebab-case

### 6. 결과 반환

발행 결과를 반환합니다. 호출측에서 후속 처리(메타 갱신 등)에 사용합니다.

| 항목 | 값 |
|------|---|
| 파일 경로 | `_posts/YYYY-MM-DD-slug.md` |
| 블로그 URL | `https://{site}/posts/{slug}/` |
| 발행 날짜 | `YYYY-MM-DD` |

## 주의사항

- **내용 검토 금지**: 콘텐츠 품질·대외비는 입력 전에 해결되어야 함
- **블로그 레포 CLAUDE.md 준수**: 포맷, 스타일링 규칙은 블로그 레포가 정의
- **형식 변환 전담**: 이 스킬의 유일한 책임은 콘텐츠를 블로그 형식으로 변환하는 것
